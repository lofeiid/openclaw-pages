交付校验

split

OpenClaw PRD 评审修复复盘 最容易出问题的地方

路由与状态

请求进错链路，或者读到了脏状态，最终产物就会和用户实际要求脱节。

  • 状态串脏会让新请求继承旧链接
  • 多路 writer 会让 HTML 和最终链接不一致
执行与验证

能跑通 helper 不等于链路可交付，最后必须把 writer 和 verification 一起锁死。

  • 多路 writer 会让 HTML 和最终链接不一致
  • helper 能运行但没验证，会把半成品发给用户
dashboard

OpenClaw PRD 评审修复复盘 需要守住的四个系统信号

这四项比“支持多少命令”更能说明一条自动化链路是不是可信。

Routing Single lane

同一类请求只能有一个 canonical path

State Fresh

新请求不能继承旧会话脏数据

Writer Trusted

只有可信 writer 能产生产物

Ready Verified

交付前要做最终 readiness 检查
